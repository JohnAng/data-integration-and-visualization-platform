# Frontend — MYE030 Data Integration & Visualization

React 19 + Vite + TypeScript application that consumes the FastAPI
backend documented in `docs/API_REFERENCE.md` and renders the
analytical pages required by the project brief.

## Stack

| Concern | Library |
|---|---|
| Build | Vite 8 |
| UI | React 19 + TypeScript |
| Routing | TanStack Router (file-based, generated tree) |
| Server state | TanStack Query |
| Styling | Tailwind CSS 4 + design tokens from `docs/design.md` |
| UI primitives | Hand-written shadcn-style components on Radix |
| Charts | Visx (`@visx/*`) |
| Icons | lucide-react |
| Schema validation | zod (route search params) |
| Testing | Vitest + Testing Library + MSW |

## Commands

```powershell
pnpm install            # install dependencies (once per environment)
pnpm dev                # dev server on :5173, proxies /api to :8000
pnpm build              # tsc + vite build
pnpm preview            # serve the production bundle locally
pnpm lint               # eslint
pnpm test               # Vitest, ~8s
pnpm test:watch         # watch mode
pnpm test:coverage      # writes coverage/ HTML report
```

The dev server expects the backend at `http://localhost:8000`. Override
with `VITE_BACKEND_URL` if it runs elsewhere.

## Routes

The application has thirteen public URLs, mapped one-to-one to files
under `src/routes/`. Parent routes that contain children are
`<Outlet />`-only layouts; the actual pages live in `*.index.tsx`
siblings.

| URL | Page | File |
|---|---|---|
| `/` | Editorial landing with KPI tiles | `routes/index.tsx` |
| `/dashboard` | At-a-glance dashboard | `routes/dashboard.tsx` |
| `/journals` | Journals list with filters | `routes/journals.index.tsx` |
| `/journals/$journalId` | Journal profile with year range | `routes/journals.$journalId.index.tsx` |
| `/journals/$journalId/articles/$articleId` | Paper details | `routes/journals.$journalId.articles.$articleId.tsx` |
| `/conferences` | Conferences list with filters | `routes/conferences.index.tsx` |
| `/conferences/$conferenceId` | Conference profile | `routes/conferences.$conferenceId.index.tsx` |
| `/conferences/$conferenceId/articles/$articleId` | Paper details | `routes/conferences.$conferenceId.articles.$articleId.tsx` |
| `/authors` | Authors list with name search | `routes/authors.index.tsx` |
| `/authors/$authorId` | Author profile | `routes/authors.$authorId.tsx` |
| `/years` | Years list (descending table) | `routes/years.index.tsx` |
| `/years/$year` | Year detail with tabs | `routes/years.$year.tsx` |
| `/charts` | Visualization playground | `routes/charts.tsx` |

The pure layout files `journals.tsx`, `journals.$journalId.tsx`,
`conferences.tsx`, `conferences.$conferenceId.tsx`, `authors.tsx`, and
`years.tsx` exist only to render an `<Outlet />` so child routes (the
profile and paper detail pages) can mount instead of the parent list.

## Testing patterns

- **Unit / component tests** colocate with their source: `Button.tsx` +
  `Button.test.tsx`. They use `renderWithQuery` from `src/test/render.tsx`
  when they need a `QueryClient` and call MSW handlers directly.
- **Route-level integration tests** also live alongside their route file
  (`routes/journals.test.tsx`). They use `renderApp(initialPath)` which
  builds a memory-history TanStack Router on top of the existing
  `routeTree.gen.ts`, so URL resolution and inter-page navigation are
  exercised end-to-end against MSW-mocked API responses.
- **API mocking** lives in `src/test/handlers.ts`. Each test can call
  `server.use(...)` to override a handler for a single test case
  (used for error-path assertions).

See `docs/design.md` for the design system and `docs/API_REFERENCE.md`
for the backend contract this app consumes.

## End-to-end tests (Playwright)

The Playwright suite under `e2e/` exercises the real app against a
live dev server + a live FastAPI backend. Three spec files cover
~50 scenarios in `chromium` (browser pinned in `playwright.config.ts`):

| Spec | Coverage |
|---|---|
| `app.spec.ts` | Landing, dashboard, list/profile/year/article navigation, year-range filter recomputation, sticky sort headers |
| `charts.spec.ts` | All chart variants in `/charts`, drag-to-zoom, legend toggle, multi-select pickers |
| `report_screenshots.spec.ts` | Generates the PNG screenshots under `deliverables/report/figures/screenshots/` |

Run with:

```powershell
# Terminal 1 — boot the MySQL container only (host-side servers below)
docker compose up -d --wait mysql_db
cd src/backend && uv run uvicorn api.main:application --port 8000
# Terminal 2
cd src/frontend && pnpm dev
# Terminal 3
cd src/frontend && pnpm exec playwright install chromium     # first-time only
pnpm exec playwright test                                    # full run
pnpm exec playwright test report_screenshots --update-snapshots  # regenerate report PNGs
```

> Alternatively, leave the full Docker stack up (`python run.py`) and
> the Playwright suite still works against the proxied frontend on
> `:5173`. The host-side `pnpm dev` is only needed when you want
> HMR while writing tests.
